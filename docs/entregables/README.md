# Entregables

Esta carpeta puede usarse para guardar los archivos finales de la materia.

Ejemplos:

- plan de pruebas en PDF.
- casos de prueba en Excel.
- reporte de bugs en Excel.
- reporte de JMeter.
- reporte de OWASP ZAP.
- checklist final de cumplimiento.
- capturas de evidencia.

Archivos principales incluidos:

- `TestPlan StudyFlow - Yepez & Santamaria.pdf`
- `Reporte Bugs StudyFlow - Santamaria & Yepez.xlsx`
- `Reporte Bugs StudyFlow - Santamaria & Yepez.pdf`
- `studyflowpro-api-test-plan.jmx`
- `ZAP-Report-study-flow-pro-kappa.vercel.app.html`
- `Checklist Calidad StudyFlow - Yepez & Santamaria.pdf`
- `Checklist Calidad StudyFlow - Yepez & Santamaria.md`

No es necesario copiar aqui el codigo de unit tests ni Cypress, porque esas
pruebas ya estan dentro de `backend/tests` y `frontend/cypress`.

La idea es mantener los entregables juntos dentro del repositorio, separados
del codigo fuente del backend y frontend.
